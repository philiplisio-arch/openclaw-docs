# OPENCLAW-ADV-016: Raw Retrieval Logging and Run Traceability Archive

## 1. CONTEXT

The D17 trust issue (D-FB-008) demonstrated that the current pipeline
preserves only the filtered retrieval package — the sources that survived
filtering and reached the agent. The full set of raw search results from
Brave and Baidu is discarded after each run.

This means that after delivery, it is not possible to answer:
- What did Brave return before filtering?
- What did Baidu return before filtering?
- Which sources were removed and why?
- Did the agent draw a fact from a source it did not cite?
- Could a stronger source have supported the claim, even if the agent
  chose a weaker one?

Without this data, post-delivery audits are incomplete and claim-source
diagnosis is limited to whatever survived the filter pass.

---

## 2. REQUIREMENT

For every run, OpenClaw must save a complete raw retrieval archive. No
raw retrieval file may be overwritten. Existing latest-run convenience
files (e.g. retrieval_package_china_monitor_001.json) may remain but
are supplemented — not replaced — by the dated archive.

### Files Required Per Run

**1. Raw Brave results**
Full unfiltered Brave return set, as received from the API.
`raw_brave_results_{client}_{YYYYMMDD}_{run_id}.json`

**2. Raw Baidu results**
Full unfiltered Baidu / SerpAPI return set, as received.
`raw_baidu_results_{client}_{YYYYMMDD}_{run_id}.json`

**3. Combined raw retrieval package**
All raw provider results merged, before any filtering.
`raw_retrieval_package_{client}_{YYYYMMDD}_{run_id}.json`

**4. Filtered retrieval package (existing — now dated)**
The package passed downstream to the agent. Already exists as a
latest-run file; must also be written to the dated archive.
`filtered_retrieval_package_{client}_{YYYYMMDD}_{run_id}.json`

**5. Filter decision log**
For every raw source, one record stating whether it was kept or removed.
`retrieval_filter_decisions_{client}_{YYYYMMDD}_{run_id}.json`

---

## 3. MINIMUM FIELDS — FILTER DECISION LOG

Each record in the filter decision log must include:

| Field | Description |
|-------|-------------|
| `run_id` | Run identifier |
| `client_id` | Client namespace |
| `provider` | Brave or Baidu |
| `provider_rank` | Original rank in provider results |
| `retrieved_at` | Timestamp of retrieval |
| `query` | Query string that produced this result |
| `title` | Article title |
| `publisher` | Publisher name |
| `domain` | Domain extracted from URL |
| `url` | Full URL |
| `date` | Article date |
| `snippet` | Full summary/snippet text |
| `raw_result_object` | Original provider result object |
| `filter_status` | `kept` or `dropped` |
| `filter_reason` | Reason if dropped (e.g. `dropped:domain_exclusion`, `warn:shell_page_keyword`) |
| `passed_to_agent` | Boolean — did this source reach the agent input? |
| `cited_in_output` | Boolean — was this source cited in final delivered output? |

---

## 4. FOLDER STRUCTURE

```
/root/openclaw_traceability/
  /{client_id}/
    /{YYYY-MM-DD}/
      raw_brave_results_{client}_{YYYYMMDD}_{run_id}.json
      raw_baidu_results_{client}_{YYYYMMDD}_{run_id}.json
      raw_retrieval_package_{client}_{YYYYMMDD}_{run_id}.json
      filtered_retrieval_package_{client}_{YYYYMMDD}_{run_id}.json
      retrieval_filter_decisions_{client}_{YYYYMMDD}_{run_id}.json
      final_output_{client}_{YYYYMMDD}_{run_id}.txt
      validation_result_{client}_{YYYYMMDD}_{run_id}.json
      claim_source_review_{client}_{YYYYMMDD}_{run_id}.json
```

`claim_source_review` is a placeholder for ADV-015 Option B output —
not required in this implementation; directory structure should include
the slot for future use.

---

## 5. STORAGE PRINCIPLE

Raw retrieval logs are immutable once written. They must not be
overwritten by subsequent runs. Each run produces its own dated,
run_id-stamped archive. Disk usage should be monitored; archival
rotation policy to be defined if storage becomes a constraint.

---

## 6. IMPLEMENTATION SCOPE (Claude Code)

Changes required:

1. **run_light_to_lark.sh** — create `/root/openclaw_traceability/{client_id}/{date}/`
   directory at run start; pass archive path to downstream scripts.

2. **package_builder.py** (or wherever Brave/Baidu results are assembled) —
   write raw provider results to archive before any filtering or merging.

3. **filter_results.py** — write filter decision log entry for every
   source processed; write filtered package to dated archive path in
   addition to existing latest-run path.

4. **run_light_to_lark.sh** (close of run) — copy final_output and
   validation_result to dated archive.

All changes require: py_compile / bash -n exit 0; end-to-end confirmation
on next run that archive directory populates correctly.
