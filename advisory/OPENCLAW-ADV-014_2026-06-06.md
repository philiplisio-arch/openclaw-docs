# OPENCLAW-ADV-014: Source Authority Filter

## 1. CONTEXT

Phase D retrieval package analysis (2026-06-06, D17) confirmed that
filter_results.py applies a single filter criterion: URL date recency
(`keep:recent_url_date`). No publisher authority gate, domain exclusion
list, or quality floor exists anywhere in the pipeline.

D17 retrieval package contained:
- h5.article.smbae.cn — WeChat article aggregator; snippet is pagination
  widget text; no substantive article content
- CCTV tv.cctv.com / tv.cctv.cn (×4) — TV video shell pages; snippets
  are trending-video sidebar navigation, not journalism
- These sources passed the freshness filter identically to primary reporting

This is a structural gap. Source authority calibration was identified in
ADV-002 (2026-05-08), ADV-012 (2026-05-14), and ADV-013 (2026-05-24) as
a Phase 7 editorial workstream. D17 confirms the gap is now causing
measurable output quality problems including confirmed fabrication (D-FB-008).

The orchestrator spec explicitly prohibits prompt-based filtering. Any
authority filter must be implemented at filter_results.py.

---

## 2. PROPOSED FIX — TWO-LAYER APPROACH

### Layer 1: Domain Exclusion List (approved — immediate)

Add a `DOMAIN_EXCLUSION` list to filter_results.py. Sources whose URL
matches an excluded domain are dropped at filter time regardless of
freshness. Applied per-client via client config flag.

Initial exclusion list (WS1 — china_monitor_001):
- `h5.article.smbae.cn` — WeChat aggregator; no article content in snippets
- `tv.cctv.com` — TV video shell pages (CP-025 already excludes for ALJ;
  extend to WS1)
- `tv.cctv.cn` — same as above

Mechanism: substring match on `result['url']` against exclusion list.
Exit with `filter_reason: "dropped:domain_exclusion"`.

Client config addition (client_config_china_monitor_001.yaml):
```yaml
domain_exclusion:
  - "h5.article.smbae.cn"
  - "tv.cctv.com"
  - "tv.cctv.cn"
```

Risk: LOW. Exclusion list is additive and reversible. WS1 currently has
4 CCTV TV shell sources in its package — removing them does not reduce
substantive content since their snippets contain no article text.

### Layer 2: Snippet Quality Floor (calibration complete — pending operator approval)

**Calibration date:** 2026-06-06 (D17 retrieval package, 12 source records)
**Field note:** Records carry a `summary` field; no `snippet` key present.
All Layer 2 filtering operates on `summary`.

#### Calibration Results

| Threshold (non-WS chars) | Pass | Fail |
|--------------------------|------|------|
| 40 | 12 | 0 |
| 60 | 12 | 0 |
| 80 | 12 | 0 |
| 100 | 11 | 1 |
| 120 | 11 | 1 |
| 150 | 5 | 7 |
| 200 | 0 | 12 |

The source failing at ≥100 is CNBC (91 chars) — a legitimate, substantive
English-language news flash. Dropping it would be a false positive.

**Shell-page flags identified (not catchable by char threshold alone):**
- 4 CCTV video shells (tv.cctv.com / tv.cctv.cn) — char counts 147–177.
  Summaries are video-player navigation widgets: 热播榜, 内容简介, pagination
  numerals. Char count is inflated by widget text; these pages pass every
  threshold ≤150. Already addressed by Layer 1 domain exclusion.
- h5.article.smbae.cn — borderline. Has real MOFCOM-quote body but tails
  into 推荐阅读 recommended-reading navigation. Already excluded by Layer 1.

**Critical finding:** In the D17 package, char count and content quality
are inversely correlated. The four shell pages (147–177) score higher than
the best real journalism (CNBC at 91). A pure char gate applied to this
package would have zero effect on the dominant failure mode.

#### Recommended Threshold

**80 non-whitespace characters** — catches truly empty stubs and
placeholder pages; retains all 12 sources in the D17 package including
the legitimate 91-char CNBC report. A threshold of ≥100 would incorrectly
drop CNBC.

#### Recommended Behavior

**Warn-only (default).** Hard-drop is operator's decision. Given that:
(a) the dominant shell-page failure mode (CCTV TV pages) is already handled
by Layer 1 domain exclusion, and (b) char count cannot distinguish
legitimate short-form journalism from navigation widgets at the counts
observed, warn-only avoids false positives while still surfacing thin
summaries for review.

Operator may upgrade to hard-drop if subsequent calibration runs confirm
that no legitimate sources fall below 80 chars.

#### False Positive Risk Assessment

**LOW at threshold 80.** No source in the D17 package falls below 80.
The closest legitimate source is CNBC at 91 chars — a single-sentence
news flash with a concrete financial figure. Risk would rise to MODERATE
if threshold is raised to 100+ (CNBC false drop) or if packages contain
other short-form legitimate sources (official statements, newswire flashes)
not represented in D17.

#### Sources Caught at Threshold 80

None from D17. At threshold 100: CNBC only (false positive risk — not
recommended). The shell pages that caused D-FB-008 (tv.cctv.com/cn,
h5.article.smbae.cn) are handled by Layer 1 and would not be caught by
any char threshold that preserves legitimate short-form content.

#### Claude Code Implementation Spec (filter_results.py)

Layer 2 gate is a companion to Layer 1 (domain exclusion). It runs after
Layer 1 on all sources that survived domain exclusion.

```python
# --- Layer 2: Snippet Quality Floor ---
SNIPPET_MIN_CHARS = int(os.environ.get('OPENCLAW_SNIPPET_MIN_CHARS', '80'))
SHELL_KEYWORDS = ['热播榜', '内容简介', '推荐阅读', '更多>']
# NOTE: '《' was removed from keyword list after full calibration review —
# it appears in legitimate Chinese journalism (e.g. IEA report citations)
# and produced a false positive on finance.china.com.cn.

def _nws_len(text):
    """Non-whitespace character count."""
    return len(text.replace(' ', '').replace('\n', '').replace('\t', ''))

def passes_quality_floor(result, threshold=SNIPPET_MIN_CHARS):
    text = result.get('summary', result.get('snippet', ''))
    char_count = _nws_len(text)
    if char_count < threshold:
        return False, 'dropped:thin_snippet', char_count
    for kw in SHELL_KEYWORDS:
        if kw in text:
            return False, 'dropped:shell_page_keyword', char_count
    return True, None, char_count
```

Environment variable `OPENCLAW_SNIPPET_MIN_CHARS` must be exported by
`load_client_config.py` from the client YAML key `snippet_min_chars`
(default: 80 if key absent). Shell keyword list is hardcoded in
`filter_results.py`; not client-configurable in Layer 2 v1.

Client config addition (client_config_china_monitor_001.yaml):
```yaml
snippet_min_chars: 80
```

`filter_results.py` must log each rejection with `result_id`,
`filter_reason`, and `char_count`. Warn-only mode: log rejection but do
not drop; include `quality_warn: true` in result metadata instead.
Hard-drop mode: remove from filtered set; log `dropped:thin_snippet` or
`dropped:shell_page_keyword`.

All changes require: py_compile exit 0; end-to-end loader confirmation;
operator approval before deployment.

**Requires operator approval before Claude Code implementation.**

## 3. WHAT THIS DOES NOT FIX

Domain exclusion and snippet quality floor prevent low-quality sources
from entering the package. They do not detect whether a source's content
supports the specific claim the agent makes. That is addressed in ADV-015.
