# OPENCLAW-ADV-014: Source Authority Filter

## 1. CONTEXT

Phase D retrieval package analysis (2026-06-06, D17) confirmed that
filter_results.py applies a single filter criterion: URL date recency
(`keep:recent_url_date`). No publisher authority gate, domain exclusion
list, or quality floor exists anywhere in the pipeline.

D17 retrieval package contained:
- h5.article.smbae.cn — WeChat article aggregator; snippet is pagination
  widget text; no substantive article content
- CCTV tv.cctv.com / tv.cctv.cn (×4) — TV video shell pages; snippets
  are trending-video sidebar navigation, not journalism
- These sources passed the freshness filter identically to primary reporting

This is a structural gap. Source authority calibration was identified in
ADV-002 (2026-05-08), ADV-012 (2026-05-14), and ADV-013 (2026-05-24) as
a Phase 7 editorial workstream. D17 confirms the gap is now causing
measurable output quality problems including confirmed fabrication (D-FB-008).

The orchestrator spec explicitly prohibits prompt-based filtering. Any
authority filter must be implemented at filter_results.py.

---

## 2. PROPOSED FIX — TWO-LAYER APPROACH

### Layer 1: Domain Exclusion List (approved — immediate)

Add a `DOMAIN_EXCLUSION` list to filter_results.py. Sources whose URL
matches an excluded domain are dropped at filter time regardless of
freshness. Applied per-client via client config flag.

Initial exclusion list (WS1 — china_monitor_001):
- `h5.article.smbae.cn` — WeChat aggregator; no article content in snippets
- `tv.cctv.com` — TV video shell pages (CP-025 already excludes for ALJ;
  extend to WS1)
- `tv.cctv.cn` — same as above

Mechanism: substring match on `result['url']` against exclusion list.
Exit with `filter_reason: "dropped:domain_exclusion"`.

Client config addition (client_config_china_monitor_001.yaml):
```yaml
domain_exclusion:
  - "h5.article.smbae.cn"
  - "tv.cctv.com"
  - "tv.cctv.cn"
```

Risk: LOW. Exclusion list is additive and reversible. WS1 currently has
4 CCTV TV shell sources in its package — removing them does not reduce
substantive content since their snippets contain no article text.

### Layer 2: Snippet Quality Floor (approved for dry-run analysis)

Add a minimum snippet length gate. Sources whose `summary` field is below
a character threshold (proposed: 80 non-whitespace characters) are dropped
with `filter_reason: "dropped:thin_snippet"`.

Rationale: legitimate article snippets contain substantive sentence
fragments. Navigation menus, pagination widgets, and TV program listings
consistently produce snippets under 80 meaningful characters.

Risk: MODERATE. Threshold requires calibration against existing package
data to avoid false positives on legitimately short but valid snippets.
Dry-run analysis across last 5 retrieval packages required before
implementation. Claude Code to pull packages and run analysis.

---

## 3. WHAT THIS DOES NOT FIX

Domain exclusion and snippet quality floor prevent low-quality sources
from entering the package. They do not detect whether a source's content
supports the specific claim the agent makes. That is addressed in ADV-015.
