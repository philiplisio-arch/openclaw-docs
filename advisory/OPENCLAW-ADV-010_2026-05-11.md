# OPENCLAW — ADVISORY MEMO
**Document:** OPENCLAW-ADV-010
**Date:** 2026-05-11
**Subject:** System Documentation Standards — Improvement Plan

---

## BACKGROUND

Post-reorganization audit (2026-05-11) graded system documentation at B−
overall. Content quality is strong (A−). The gaps are in convention
discipline: inconsistent naming, two version numbering formats in use,
changelog practice applied to only one document, four stale filename dates
identified, and no index for the old/ archive. None of these are urgent.
All are addressable in one focused session without touching any production
documents or pipeline code.

---

## PROPOSED IMPROVEMENTS (in priority order)

### IMPROVEMENT 1 — Standardize version numbering
**Priority:** High
**Scope:** Frontmatter in all active documents

Two formats currently in use:
- Numeric (e.g., "2.3") — used by Operating Protocol
- Date-as-version (e.g., "5.11.26") — used by most other docs

Proposed rule: date-as-version (MM.DD.YY) is the standard for all
documents except the Operating Protocol, which retains numeric versioning
due to its formal change-control history. This matches how most docs
already work and requires no retroactive changes to the Operating Protocol.

Action: Add `version: MM.DD.YY` to frontmatter of any active document
that currently lacks it. No content changes.

---

### IMPROVEMENT 2 — Add changelog footers to top-5 frequently updated docs
**Priority:** High
**Scope:** 5 documents only

The Operating Protocol's changelog format is the model. A one or two-line
entry per update is sufficient — date, what changed, who approved.

Documents to receive changelog footers:
```
03_Issues Log
04_DAILY_STATUS
06_PHASE_GATE_CHECKLIST
governance/OPENCLAW-BRAIN-LITE-DESIGN
governance/OPENCLAW-TEST-HARNESS-DESIGN
```

Format (append to bottom of each document):
```
---
CHANGE LOG
MM.DD.YY — [what changed] — operator approved
```

Action: Add footer section with existing change history reconstructed
from session records where known. Future updates append one line per
change.

Requires separate operator approval as a locked-document touch. Implement
in Session B only.

### IMPROVEMENT 5 — Create old/ archive index
**Priority:** Low
**Scope:** New file only — old/ contents not modified

Create `old/ARCHIVE_INDEX.md` listing all files in old/ with date and
one-line description. Append-only going forward. Allows historical
versions to be found without browsing ~90 files.

Format:
```
| Filename | Superseded Date | Superseded By |
|---|---|---|
| 04_DAILY_STATUS (5.8.26).md | 2026-05-11 | 04_DAILY_STATUS (5.11.26).md |
```

---

### IMPROVEMENT 6 — Standardize cross-document citation format
**Priority:** Low
**Scope:** Style rule only — no document edits required immediately

Currently documents reference each other using a mix of:
- document_id (OPENCLAW-OPS-001)
- filename (05_OPERATING_PROTOCOL)
- informal name (Operating Protocol)

Proposed rule: always cite by document_id in formal references within
governance and spec documents. Informal names acceptable in operational
docs (Daily Status, Issues Log). Apply going forward — no retroactive
edits required.
