# OPENCLAW-ADV-017: CEO-Controlled Product Quality Workflow

## 1. PURPOSE

This advisory establishes the governing workflow model for OpenClaw from
2026-06-06 forward. It replaces the engineering-led model (build → run →
validate → deliver) with a five-layer product-quality model that separates
system reliability, citation structure, source quality, claim support, and
client usefulness.

Trigger: D-FB-008 (D17, 2026-06-06) — a real citation was attached to a
claim the cited source did not support. The validator passed. The delivery
was sent, then retracted. The workflow model that allowed this must change.

## 3. FIVE-LAYER OPERATING MODEL

All OpenClaw deliveries are evaluated against five layers in sequence.
A delivery is not counted as clean unless all five layers are satisfied.

### Layer 1 — System Run Check
*Did the system run successfully?*
Confirms: run completed; retrieval happened; brief produced; delivery held
or sent as intended.
- Green: ran normally
- Yellow: ran with warning
- Red: failed or incomplete

### Layer 2 — Citation Check
*Do citations point to real retrieved sources?*
This is what the current validator does. It confirms citation IDs exist and
are structurally valid. It does NOT confirm claim support.
- Green: all citation IDs valid
- Yellow: some issues but output may be usable after review
- Red: citation integrity failure

### Layer 3 — Source Quality Check
*Are the sources good enough to support client-facing claims?*
Builds on existing work: ADV-014 (domain exclusion, snippet quality floor),
CP-020 (source taxonomy, freshness labels). Does not replace these — extends
them into an operator-facing view. Source categories and taxonomy must be
reconciled with CP-020 to avoid two competing classification systems.

Source categories:
1. Official / regulatory source
2. Top-tier media
3. Sector / trade source
4. Local / low-authority source
5. Poor-quality / non-article source (exclude or flag)

Rule: weak sources can support weak signals; they cannot support high-stakes
claims (government action, tariffs, sanctions, regulatory decisions, diplomatic
visits, direct quotes, financial figures, investigations, major transactions,
named officials).

- Green: strong source base
- Yellow: mixed quality
- Orange: weak sources present, review required
- Red: poor-quality sources carrying major claims

### Layer 4 — Claim Support Check
*Does the cited source actually support the sentence?*
This layer operationalizes the existing claim-support workstream (ADV-015).
The manual spot-check control (Option D) is already active. The automated
alignment check spec (Option B) is pending. This layer does not restart that
work — it places it in the governing workflow.

Every Executive Take bullet must be checked against its cited source:
- Does the source mention the main person, company, country, or organization?
- Does the source include the quoted phrase, number, date, or policy term?
- If the bullet combines two facts, are both supported by citations?
- Is the source strong enough for the level of claim?
- Is wording too strong for available evidence?

Review table format:

| Executive Take | Support Status | Reason |
|----------------|---------------|--------|
| Bullet 1 | Supported | Source mentions named company and policy action |
| Bullet 2 | Needs review | Source supports trade issue but not quoted language |
| Bullet 3 | Hold | Cited source does not mention named official |

- Green: source clearly supports claim
- Yellow: partial support / wording needs softening
- Orange: source mismatch likely
- Red: claim unsupported or wrongly cited

### Layer 5 — Client Usefulness Review
*Is this actually useful to a communications executive?*
Operator/product layer. A brief can be technically clean but not client-grade.
Assess: main point clear; highlights what matters; not repetitive; avoids
generic language; sources strong enough; advisory implication useful; would
a senior PR person want this before a client meeting.

- Green: client-ready
- Yellow: useful but needs review
- Orange: not yet client-grade
- Red: do not send

## 4. DAILY WORKFLOW

Six-step sequence for every run:

1. **System produces brief** — automated run generates draft, source list,
   citation check result, source quality summary, claim-support warnings if
   available.
2. **Automated technical check** — pipeline completion, citation existence,
   no fake IDs, delivery held or sent as intended.
3. **Source quality review** — strong/weak source count, excluded sources,
   non-article pages detected, major claims on weak sources flagged.
4. **Claim support review** — each Executive Take bullet reviewed against
   its cited source; support status assigned.
5. **Operator business review** — accuracy, usefulness, language calibration,
   should this count as a clean delivery?
6. **Delivery / hold decision** — Send / Send after light edits / Hold for
   review / Do not send and reset count.

## 7. TRUST CHECKLIST (per delivery)

- Did the system run successfully?
- Are all citations structurally valid?
- Are the cited sources credible enough?
- Does each Executive Take claim match its cited source?
- Are quotes, numbers, and named officials supported?
- Are weak sources clearly flagged or excluded?
- Is the advisory language appropriately cautious?
- Is the brief useful to a communications client?
- Should this count as a clean delivery?

---

## 8. GOVERNANCE RULES

1. Validator green does not equal client-ready.
2. Clean delivery = technically clean AND product-clean (both confirmed).
3. Weak sources cannot carry strong claims.
4. High-risk claims need high-quality source support.
5. Manual review comes before automation.
6. No expansion until trust repair is contained.
7. Existing work must be completed, not duplicated — Layers 3 and 4 build
   on ADV-014, CP-020, and ADV-015; they do not restart them.

---

## 9. CEO-LEVEL SUCCESS METRICS

1. Clean deliveries confirmed (technically + product-clean)
2. Executive Take support rate (% of top bullets clearly source-supported)
3. Weak-source exposure (frequency in top claims)
4. Unsupported-claim flags (held or revised due to weak support)
5. Source quality mix (official / top-tier / sector / weak share)
6. Operator review burden (manual review still required)
7. Client usefulness score
