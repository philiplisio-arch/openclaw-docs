---

## 1. Current Framing

AI Brain has evolved from a GitHub + Obsidian memory question into a broader architecture for an AI Chief of Staff / external executive function.

The core purpose is to help Philip maintain situational awareness across projects, preserve valuable conversational context, surface decisions, and convert informal AI conversations into structured, executable work.

The key refinement from Session 001 is that GitHub is not only documentation. It is also the continuity layer that allows future AI instances to reconstruct the project state at the start of each session.

---

## 2. Current Working Model

```text
Conversation streams
  -> AI Chief of Staff
  -> Persistent entity/project/principle model
  -> Governance engine
  -> Execution layer
  -> Dashboard / briefing layer
  -> Checkpoint / repository synchronization layer
```

---

## 3. Active Design Themes

- Conversation as the primary interface
- AI Chief of Staff, not just memory
- External executive function
- Project-state awareness
- Entity model for projects, people, principles, decisions, and ideas
- Trust and validation as constitutional principles
- Governance thresholds for autonomous action
- GitHub as official record and boot loader
- Obsidian as associative memory
- Dashboard as cockpit, not source of truth
- Documentation reflects understanding, not chronology
- Checkpoint is incomplete until GitHub is synchronized
- Execution beats explanation

## 5. Current Priority

The next design domain is **Modeling the Decision Maker**.

The goal is to understand Philip's operating model well enough that AI Brain can eventually make recommendations consistent with his long-term priorities, judgment style, and way of thinking—not merely based on stored facts or task lists.

Questions to explore:

1. How does Philip recognize a good idea?
2. How do projects mature, stall, or die?
3. What causes Philip to lose track?
4. What makes an AI recommendation trustworthy?
5. How does Philip balance exploration versus execution?
6. What would count as AI Brain misunderstanding him?

---

## 6. Current Process Status

Session 001 produced a useful failure: the AI described the need for repository updates but initially did not execute them.

This exposed a process requirement:

> A checkpoint is not complete until the repository has actually been updated, or the failure to update has been explicitly recorded.

This process rule has now been incorporated into the architecture.
