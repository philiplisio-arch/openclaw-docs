# AI Brain Architecture Process

Status: Working draft
Created: 2026-06-30
Purpose: Define the working process for AI Brain architecture sessions.

---

## 1. Process Name

Architecture Discovery Process.

---

## 2. Core Loop

```text
Discover -> Challenge -> Consolidate -> Synchronize -> Checkpoint -> Continue
```

---

## 3. Session Modes

### Design Discovery

Used for open-ended questioning, concept development, and architectural exploration.

### Reality Check

Used to test whether the architecture can be built with current tools and workflows.

### Editorial Consolidation

Used to decide what changed, what belongs in the official record, and what should be pruned or clarified.

---

## 4. Checkpoint Definition

A checkpoint is complete only when:

1. The session has been reviewed.
2. Durable insights have been identified.
3. Relevant GitHub documents have been updated.
4. A session log entry has been created or updated.
5. The next session starting point is clear.

If repository synchronization fails, the checkpoint status is pending.

---

## 5. Startup Protocol

At the start of each AI Brain architecture session, read these files in order:

1. `AI_BRAIN_CURRENT_TRUTH.md`
2. `AI_BRAIN_STATE_OF_PLAY.md`
3. `AI_BRAIN_DECISION_LOG.md`
4. `AI_BRAIN_OPEN_QUESTIONS.md`
5. `AI_BRAIN_ARCHITECTURE_PROCESS.md`
6. `AI_BRAIN_ARCHITECTURE_MANIFESTO.md` only if deeper context is needed

Then continue from the next session topic.

---

## 6. Execution Rule

When an agreed action requires a tool and the tool is available, execute first and explain afterward.

If a tool is unavailable or fails, state the actual failure directly.
