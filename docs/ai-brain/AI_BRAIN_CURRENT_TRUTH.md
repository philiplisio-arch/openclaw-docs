## 1. What We Are Building

AI Brain is an AI Chief of Staff / external executive function for Philip.

It is designed to help him maintain situational awareness across many parallel projects, preserve valuable conversational context, surface decisions, connect ideas across domains, and convert informal AI conversations into coherent executable work.

It is not primarily a note-taking system, memory archive, task manager, or automation platform.

---

## 2. Core Architectural Shift

The project began as a question about how Obsidian and GitHub should work together.

It has now shifted to a higher-level architecture:

```text
From: GitHub + Obsidian memory architecture
To: AI Chief of Staff / external executive function / human-AI operating system
```

Obsidian and GitHub remain important, but they are implementation layers beneath a broader operating model.

---

## 3. Current Working Model

```text
Philip
  -> Conversation streams
  -> AI Chief of Staff
  -> Persistent entity/project/principle model
  -> Governance engine
  -> Execution layer
  -> Dashboard / briefing layer
  -> Checkpoint / repository synchronization layer
```

---

## 4. Accepted Core Principles

- Conversation is the primary interface.
- Understanding is more valuable than storage.
- Memory is curated, not accumulated.
- Projects emerge from repeated thought.
- The AI should exercise judgment, not merely retrieval.
- Trust and validation are constitutional principles.
- Autonomy requires transparency and approval thresholds.
- Documentation reflects understanding, not chronology.
- The AI has an editorial responsibility, not merely an archival one.
- Execution beats explanation.
- A checkpoint is incomplete until the repository is synchronized.
- GitHub is both repository of truth and boot loader.

## 6. Current Process Rule

Every Architecture Discovery session should end with a checkpoint.

A checkpoint requires:

1. Editorial review
2. GitHub document updates
3. Session log update
4. Next-session starting point
5. Explicit checkpoint completion statement

If GitHub synchronization does not occur, the checkpoint is pending, not complete.
