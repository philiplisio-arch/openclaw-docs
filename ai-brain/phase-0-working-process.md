# AI Brain — Short-Term Working Process (Phase 0)

**Status:** Active  
**Source Document:** Yes  
**Repository Location:** `ai-brain/phase-0-working-process.md`

## Objective

The purpose of Phase 0 is **not** to design the AI Brain.

The purpose is to understand:

1. What the AI Brain should become.
2. What already exists today.
3. What we can learn before making architectural decisions.

No implementation decisions should be considered final during Phase 0.

---

# Two Parallel Conversations

## Conversation A — Vision (Phase 0.1)

**Purpose**

Discover the desired behavior and long-term architecture of the AI Brain.

Topics include:

- Long-term goals
- Philosophy
- Human–AI relationship
- Memory concepts
- Project organization
- Desired workflows
- Ideas that occur during everyday work
- Design principles

### Rules

- Conversation should remain natural.
- Ideas are explored, not immediately implemented.
- Every significant session produces meeting notes.
- Every few sessions produce updated architecture documents.

Primary question:

> "What should the AI Brain become?"

---

## Conversation B — Current State Mapping (Phase 0.2)

**Purpose**

Create a factual description of today's ecosystem.

Topics include:

- GitHub repositories
- Claude Code
- VPS
- OpenClaw Platform
- WS1
- WS2
- APIs
- Automations
- Existing documentation
- Memory mechanisms
- Current workflows

### Rules

- Observe before optimizing.
- Record facts before proposing improvements.
- Avoid redesign discussions unless explicitly requested.

Primary question:

> "What exists today?"

# Source Document Role

This document is the short-term operating source for Phase 0.

Until replaced or revised, it defines how the AI Brain project should proceed in the near term:

- This conversation is the Vision / Phase 0.1 thread.
- A separate conversation should be used for Current State Mapping / Phase 0.2.
- GitHub should hold the durable record of roadmap, process, notes, and decisions.
- No implementation architecture should be finalized until Phase 0 produces enough evidence.

---

# Completion Criteria

Phase 0 is complete when we can confidently answer:

1. What are we trying to build?
2. What exists today?
3. What have we learned from existing AI practice?
4. What architectural problems remain to be solved?

Only then does detailed architecture design begin.
